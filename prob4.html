Entity & Attribute Definitions (sample)

EMPLOYEE — Supertype (represents every person employed by the organization)

  • EmployeeNumber (PK) — integer, unique identifier for employee. Sample value: 1001.
  • EmployeeName — varchar(100). Sample: Maria Santos.
  • Address — varchar(150). Sample: Brgy. San Roque, Zamboanga City.
  • DateHired — date. Sample: 2019-06-15.
  • EmployeeType — discriminator; possible values: Hourly, Salaried, Consultant.

HOURLY_EMPLOYEE — Subtype (attributes specific to hourly-paid staff)

  • EmployeeNumber — PK and FK referencing EMPLOYEE(EmployeeNumber).
  • HourlyRate — decimal(7,2). Sample: 150.00.
  • OvertimeRate — decimal(7,2). Sample: 225.00.

SALARIED_EMPLOYEE — Subtype (attributes for salaried employees)

  • EmployeeNumber — PK and FK referencing EMPLOYEE.
  • AnnualSalary — decimal(12,2). Sample: 450000.00.
  • StockOption — integer (number of options). Sample: 1000.

CONSULTANT — Subtype (contract-based staff)

  • EmployeeNumber — PK and FK referencing EMPLOYEE.
  • ContractNumber — varchar(30). Sample: CN-2023-045.
  • BillingRate — decimal(8,2). Sample: 900.00 (per day/hour depending on business rule).

Relationships & Constraints

  • Specialization (EMPLOYEE → {HOURLY_EMPLOYEE, SALARIED_EMPLOYEE, CONSULTANT}) — supertype/subtype relationship. Disjoint (an employee instance belongs to exactly one subtype) and total (every employee is one of the subtypes). The EmployeeType attribute on EMPLOYEE acts as the subtype discriminator.
  • Attribute inheritance: All subtype instances inherit EMPLOYEE attributes (EmployeeName, Address, DateHired).
  • Referential integrity: Subtype table PKs are FKs referencing EMPLOYEE(EmployeeNumber); cascade delete or restrict depending on business rule.

Mapping to Relations (sample DDL)

CREATE TABLE Employee (
  EmployeeNumber INT PRIMARY KEY,
  EmployeeName VARCHAR(100),
  Address VARCHAR(150),
  DateHired DATE,
  EmployeeType VARCHAR(20)  -- discriminator
);

CREATE TABLE HourlyEmployee (
  EmployeeNumber INT PRIMARY KEY,
  HourlyRate DECIMAL(7,2),
  OvertimeRate DECIMAL(7,2),
  FOREIGN KEY (EmployeeNumber) REFERENCES Employee(EmployeeNumber)
);

CREATE TABLE SalariedEmployee (
  EmployeeNumber INT PRIMARY KEY,
  AnnualSalary DECIMAL(12,2),
  StockOption INT,
  FOREIGN KEY (EmployeeNumber) REFERENCES Employee(EmployeeNumber)
);

CREATE TABLE Consultant (
  EmployeeNumber INT PRIMARY KEY,
  ContractNumber VARCHAR(30),
  BillingRate DECIMAL(8,2),
  FOREIGN KEY (EmployeeNumber) REFERENCES Employee(EmployeeNumber)
);
